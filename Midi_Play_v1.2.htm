【 Note of Program 】Check the following items

■ 01. Ensure that all waiting loop (or loop) in program are under control and each one of
         them would not be an endless loop, even if SRAM have wrong data.

         保證程式中所有的迴圈是依據控制執行，並且不會造成死迴圈，甚至連SRAM錯誤也不會死迴圈。


■ 02. Suggest a power up delay is used, time of 0.1 sec ~ 0.3 sec is suggested.
         建議有一個power up delay，時間建議0.1~0.3秒。


■ 03. Ensure all control-registers/control-variables are initiated in reset or main loop.
         保證所有的控制暫存器或控制變數在RESET或main loop有做初值化。


■ 04. Suggest, if IO control or other registers is not changed during this application,
         these registers should be set in main loop. For example, Timer,
         interrupt enable, Analog controls...

         建議假如IO的控制或其他的暫存器在應用上是沒有使用的，那麼在main loop必須設定初值。        

         例如 : Timer, interrupt enable, IO controls, Analog controls,………。


■ 05. Suggest watch dog timer is cleared in main loop only, do not clear it in interrupt service routine.
         建議在main loop有唯一的清除watch dog timer動作，不要在中斷程序裏做清除的動作。


■ 06. Suggest, before enter halt mode, discharge all wake up related input ports to proper
         state. (set to output 0 and back to input mode, this charge could cause false trigger) And
         all output port value shall be programmed.
         建議在進入HALT模式之前，必須設定所有的wakeup輸入腳到適合的狀態(例如:設定輸出0，

         然後設為輸入模式，這動作可以避免錯誤觸發)，並且所有輸出腳狀態必須依程序設定。


■ 07. Suggest, DAC data is set to "00" when initial or DAC is not used.

         建議在initial或沒有使用DAC的時候，必須設定DAC資料為0。

■ 08. Suggest, if application uses input pin to select mode or model, pull down resistor is not used.

         (current will result if connected to VDD )

         建議假如應用上是使用輸入腳做模式或產品選擇，那麼必須不要使用pull down電阻。

         (假如接到VDD會耗電流)。


■ 09. Suggest, all input pins shall have internal pull down or have external VDD (or VSS) connection.
         建議所有的輸入腳必須有內部的pull down或外部連接到VDD(或VSS) 。


■ 10. Suggest, after these I/O ports A,B,C and D are switched from output mode to input mode,
          please note the effect of charge on wire, some delay may be needed on some applications.

          建議IO的控制由輸出切換到輸入後，需注意信號腳儲存電荷效應，讀此port之前需視應用

          決定是否加延遲時間。


■ 11. Suggest, if SRAM is used both in interrupt and main program, INT is disabled when read/write

         SRAMs in main program.

         (Note：One NOP instruction is needed right after interrupt disable instruction.)
         建議假如某些SRAM在中斷程序及main 程序均有使用，那麼在main程序中使用這些SRAM

         做讀寫之前必須要先關掉INT。(注意：在中斷關掉之後需要加一個NOP)。


■ 12. Ensure, unexpected key code is processed or check that this will not cause system dead lock.

         保證有一個無效鍵值處理程序，或者保證這個無效鍵值不會造成系統有非預期程序發生。


■ 13. Ensure that all external wakeup and INT input signal is processed by the debounce procedure,

         which avoid error trigger.

         保證所有的外部wakeup與INT輸入腳有經過防止彈跳處理程序，以避免誤觸發。


■ 14. Ensure that a system reliability check procedure exist in main loop. The DSP chip will store

         the value 0007H and 0006H to the SRAM address 0000H and 0001H when system boot normally.
         保證main loop存在系統檢查碼的處理程序。DSP在開機時會將0007H,0006H這兩個系統檢查碼

         分別寫入SRAM位址0000H,0001H中。

 

■ 15. Attention that the ar.b0 value is useless when the 'slc ar,2'-type instructions are operated.

         注意使用 'slc ar,2' 這類指令時，ar.b0的值是unknown的。
